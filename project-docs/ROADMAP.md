# Roadmap

План работ по проекту `primerkakoles-app`.

Правило ведения: сюда добавляем будущие задачи, приоритеты и открытые вопросы. Уже сделанные изменения переносим или дублируем в `project-docs/PROJECT_LOG.md`.

## Ближайший Приоритет

- Развернуть код `cloudflare/primerkakoles-edge.js` в Cloudflare Worker `primerkakoles-edge`.
- Проверить `/api/profile`, `/api/gallery`, `/api/my-generations` и `/media/*` на production-домене.
- Добавить явную и понятную обработку отсутствующих env-переменных.
- Проверить production build в среде с установленным Node/npm.
- Убрать или документировать `vercel.json.bak`.
- Проверить webhook на защиту от прямого несанкционированного использования.
- Подготовить базовый checklist ручного тестирования перед деплоем.
- В Supabase Auth проверить настройку email confirmation: для текущего сценария она должна быть отключена.
- Продумать и включить защиту от массовых регистраций.

## UX И Продукт

- Улучшить UX авторизации и ошибок email/password.
- Проверить поведение автозаполнения телефона на мобильных устройствах.
- Добавить более понятные состояния ошибок webhook.
- Добавить подсказки по требованиям к фото автомобиля и диска.
- Улучшить пустые состояния личной и публичной галереи при необходимости.
- Позже добавить восстановление пароля через email.

## Генерация И Загрузка Фото

- Добавить валидацию размера и веса загружаемых изображений.
- Добавить более дружелюбное сообщение при timeout генерации.
- Проверить сценарий пользователя без оставшихся генераций.
- Проверить сценарий нового пользователя с `0` доступных примерок в кабинете.
- Проверить, что миграция `20260505_pr8_reset_web_starter_grants.sql` сняла старый баланс `1 из 1` у тестового аккаунта.
- Проверить сценарий истекшей Supabase-сессии при генерации.
- Исправить возможный memory leak object URL при замене файла в `PhotoUpload`.
- Сделать чтобы фотографии которые хранятся в supabase, открывались в рф, придумать обход чтобы работало без vpn

## Архитектура И Код

- Вынести общий gallery/preview компонент из `GalleryPage` и `MyGenerationsPage`, если галереи продолжат развиваться.
- Рассмотреть общий helper для форматирования дат в галереях.
- Рассмотреть общий helper для форматирования и нормализации российского телефона.
- Добавить базовую проверку качества кода, если проект продолжит расти: lint/format/build-check.

## Supabase И Безопасность

- Проверить RLS-политики на таблицах `users` и `generations`.
- Проверить, что публичная галерея не раскрывает приватные данные.
- Проверить, что пользователь видит только свои личные генерации на `/my`.
- Проверить, что webhook валидирует Supabase access token.
- Проверить, где именно и кем обновляется `generations_used`.
- Рассмотреть Supabase Captcha для signup: hCaptcha или Cloudflare Turnstile.
- Ограничить массовые регистрации на уровне Supabase Auth rate limits и/или captcha.
- Не полагаться на frontend-only защиту от массовых регистраций.

## Деплой И QA

- Проверить прямые URL GitHub Pages: `/try`, `/gallery`, `/my`, `/login`, `/cabinet`.
- Проверить, что `public/CNAME` попадает в production build.
- Проверить GitHub Actions build после каждого изменения зависимостей или workflow.
- Сформировать короткий ручной smoke-test перед push/deploy.

## Открытые Вопросы

- Нужно ли полностью удалить `vercel.json.bak` или оставить как исторический артефакт?
- Какой контакт поддержки должен быть основным для восстановления доступа?
- Нужен ли Telegram Login в web UI с учетом блокировок в РФ?
- Где должна жить бизнес-логика покупки/пополнения генераций?
- Реализовать оплату/покупку генераций для кнопки `Купить генерации` в личном кабинете.
- Нужна ли пагинация или infinite scroll для галерей после роста данных?
